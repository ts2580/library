# Bookshelf Improvement Plan

## 진행 원칙

- 데이터 정합성, 보안, 운영 안정성처럼 장애나 노출로 이어질 수 있는 항목을 먼저 처리한다.
- 기존 SQLite 데이터와 호환되지 않는 제약은 즉시 강제하지 않고, 검증과 애플리케이션 레벨 방어를 먼저 둔다.
- 각 묶음은 테스트 또는 스크립트로 확인 가능한 상태까지 진행한다.

## 완료한 우선 개선

- 재고 일괄 갱신에서 조회 결과가 빈 목록이면 기존 지점 재고를 삭제하도록 변경했다.
- 재고 교체는 삭제와 삽입을 하나의 트랜잭션 메서드로 처리한다.
- 상품 등록을 트랜잭션으로 묶고, `isbn13` 중복은 SQLite 유니크 인덱스로 방어한다.
- 동일한 `book + volume` 조합은 한정판, 외전, 단편집, 상/하권 등을 표현하기 위해 허용한다.
- 알라딘 API 키 기본값을 제거하고, 키가 없으면 외부 API 호출이 안전하게 빈 결과로 실패하도록 했다.
- 알라딘 API 실패 로그에서 `ttbkey` 값을 마스킹하고, 외부 호출 타임아웃을 명시했다.
- remember-me 서명 키를 환경변수로 주입하도록 하고, 자동 remember-me 대신 로그인 화면의 명시적 체크박스로 바꿨다.
- Docker 빌드 컨텍스트에서 로컬 DB, 환경 파일, 빌드 산출물을 제외하도록 `.dockerignore`를 추가했다.
- npm lockfile을 추적 대상에 포함하고, Docker와 `run.sh`가 lockfile 기반 설치를 우선 사용하도록 정리했다.
- Chart.js CDN 의존을 제거하고 npm lockfile에 고정된 로컬 정적 자산으로 서빙하도록 바꿨다.

## 완료한 추가 개선

- 회원가입 후 자동 로그인에서 Spring Security 세션 키 상수를 사용하고 `ROLE_USER` 권한을 명시했다.
- 상품 검색 화면에서 쓰지 않는 전체 책 목록 조회를 제거해 `/products` 렌더링 비용을 줄였다.
- 현재 사용자 모델 주입을 공통 `ControllerAdvice`로 옮겨 각 컨트롤러의 반복 호출을 줄였다.
- Spring Security가 보호하는 페이지/폼 경로의 수동 로그인 체크를 제거하고, JSON API의 명시적 401 처리만 유지했다.
- 운영 profile에서는 Thymeleaf cache를 켜고 SQL init을 끄도록 설정을 분리했다.
- SQLite 검증 스크립트가 중복 `isbn13` 방어를 확인하도록 보강했다.
- 현재 코드와 맞지 않는 오래된 `blko-*` 디자인 토큰 리포트와 대형 refactor diff 산출물을 제거했다.

## 보류/제외

- 기존 SQLite DB의 `(book, volume)` 중복 데이터는 의도된 데이터 표현이므로 자동 정리 대상에서 제외한다.
- 데이터 정제는 별도 수동 작업으로 진행하며, 애플리케이션은 `isbn13` 중복만 강제한다.
- 따라서 `(book, volume)` 유니크 제약 승격은 진행하지 않는다.
